# Telegram-бот: вакансии на сайте

Сайт читает вакансии из файла `data/vacancies.json`, из **`GET /api/vacancies`** (Vite), из **`VITE_VACANCIES_API_URL`** или из **Supabase** (см. [SUPABASE.md](SUPABASE.md)) — в зависимости от переменных окружения. Отдельный процесс **`npm run bot`** подключается к Telegram и позволяет админам добавлять и удалять записи; при **`SUPABASE_URL` + `SUPABASE_SERVICE_ROLE_KEY`** бот пишет в таблицу `vacancies` вместо файла.

## Переменные окружения

Скопируйте [.env.example](.env.example) в `.env.local` (файл с `*.local` не коммитится).

| Переменная | Где нужна | Описание |
|------------|-----------|----------|
| `TELEGRAM_BOT_TOKEN` | Vite (`/api/lead`), бот | Токен от [@BotFather](https://t.me/BotFather). Один бот может и слать лиды с формы (HTTP `sendMessage`), и работать в long polling для команд. |
| `TELEGRAM_CHAT_ID` | Только Vite | Куда слать заявки с формы (числовой id чата). |
| `TELEGRAM_ADMIN_IDS` | Только бот | Список **числовых Telegram user id** админов через запятую. Только они могут использовать команды бота. |
| `VACANCIES_FILE` | Vite + бот | Путь к JSON (по умолчанию `data/vacancies.json` от корня проекта). Не используется, если включён Supabase. |
| `VITE_VACANCIES_API_URL` | Сборка фронта | Если сайт на статике без Node — полный URL к вашему `GET .../vacancies`. |
| `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` | Бот (запись), скрипты | Хранение вакансий в Postgres без VPS; см. [SUPABASE.md](SUPABASE.md). |

## Как узнать свой Telegram user id

1. Напишите боту [@userinfobot](https://t.me/userinfobot) или [@getidsbot](https://t.me/getidsbot) — он пришлёт число `Id`.
2. Добавьте это число в `TELEGRAM_ADMIN_IDS` в `.env.local`.

## Подготовка папки данных

```bash
mkdir -p data
```

Файл `data/vacancies.json` создаётся автоматически при первой записи. В репозитории он в `.gitignore`, чтобы не коммитить внутренние объявления.

Чтобы подставить **старую вакансию с сайта** (Affiliate Manager и тексты из локалей), скопируйте шаблон:

```bash
cp data/vacancies.seed.json data/vacancies.json
```

(на Windows в PowerShell: `Copy-Item data/vacancies.seed.json data/vacancies.json`)

## Локальный запуск

В **двух** терминалах из корня проекта:

```bash
npm run dev
```

```bash
npm run bot
```

Откройте сайт, блок «Вакансии» запросит `GET /api/vacancies`. В боте (тот же токен, что в `.env.local`) выполняйте команды от аккаунта, чей id в `TELEGRAM_ADMIN_IDS`.

## Команды бота

- **`/start`** — краткая справка.
- **`/list`** — список вакансий (короткий заголовок EN, id, дата) и кнопки **«Удалить N»**.
- **Удаление** — после кнопки бот спросит подтверждение (**Да, удалить** / **Отмена**).
- **`/show <uuid>`** — полный текст всех заполненных полей (EN / RU / UK).
- **`/add`** — мастер добавления:
  1. Заголовок **на английском** (обязательно).
  2. Требования **на английском** (обязательно, можно несколько строк).
  3. Заголовок **RU** или слова **`skip`** или **`скип`** обычным текстом (без `/`, иначе Telegram воспринимает как команду).
  4. Если RU-заголовок был — требования **RU** (обязательно).
  5. Заголовок **UK** или снова **`skip`** или **`скип`** тем же способом.
  6. Если UK-заголовок был — требования **UK** (обязательно).
- **`/cancel`** — сброс сценария `/add`.
- На шагах RU/UK заголовка команда **`/skip`** тоже сработает (на случай, если клиент отправил её как команду); в подсказках бота указаны обычные слова **skip** / **скип** без слэша.

На сайте для языка интерфейса используется цепочка: текущая локаль → при отсутствии перевода показывается **английский** текст.

## Продакшен

- **Статический хостинг без Node** (`dist` на CDN): `GET /api/vacancies` **недоступен**. Задайте `VITE_VACANCIES_API_URL` на ваш backend, который отдаёт тот же JSON-массив, что и локальный API, и держите тот же файл или БД на сервере рядом с ботом.
- **Один VPS**: поднимите сайт + прокси на Node (или оставьте Vite SSR/Express) так, чтобы `/api/vacancies` и `/api/lead` работали; бот и сайт должны видеть **один и тот же** `VACANCIES_FILE`.

## Формат JSON (справка)

Массив объектов:

```json
[
  {
    "id": "uuid",
    "createdAt": "2026-04-04T12:00:00.000Z",
    "title": { "en": "…", "ru": "…", "uk": "…" },
    "requirements": { "en": "…", "ru": "…", "uk": "…" }
  }
]
```

Поля `ru` и `uk` опциональны; **`title.en`** и **`requirements.en`** обязательны.

## Безопасность

- Не публикуйте `TELEGRAM_BOT_TOKEN`.
- Ограничивайте доступ только `TELEGRAM_ADMIN_IDS`.
- По желанию заведите **отдельного** бота только для админских команд и оставьте текущий токен только для отправки лидов — тогда в коде нужно два токена (отдельная переменная для бота вакансий). Сейчас используется один `TELEGRAM_BOT_TOKEN` для обоих сценариев.
